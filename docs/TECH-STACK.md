# AurumTrace — Technology Stack & DevOps

AI AML-compliance application for UAE gold / precious-metals dealers (DPMS / goAML).
This document lists the full technology stack and the DevOps / deployment work, with
exact dependency versions.

---

## 1. Languages & Core

- **TypeScript** 5.6.3 (strict mode, no `any`) — across all packages
- **Node.js** 20 LTS (production pinned to 20.18.1)
- **SQL** (PostgreSQL dialect)

## 2. Monorepo & Tooling

- **pnpm** 9.15.0 workspaces (via Corepack) — package manager
- **Turborepo** ^2.3.3 — monorepo task orchestration
- Workspace layout:
  - `apps/web` — Next.js frontend
  - `apps/api` — NestJS backend
  - `packages/shared` — shared Zod schemas & types
  - `packages/db` — Postgres migrations + RLS
  - `packages/integrations` — third-party provider adapters

## 3. Frontend (`apps/web`)

- **Next.js** 14.2.21 — App Router, Server Components, Server Actions
- **React** 18.3.1 / **react-dom** 18.3.1
- **Tailwind CSS** 3.4.17 — mapped to CSS-variable design tokens
- **PostCSS** 8.4.49 / **Autoprefixer** 10.4.20
- **ESLint** 8.57.1 / **eslint-config-next** 14.2.21
- Internationalisation — English + Arabic, cookie-driven `<html dir>` RTL switching
- Custom AurumTrace design system (brand tokens, six-stage case wizard, dashboard, audit views)

## 4. Backend (`apps/api`)

- **NestJS** 10.4.15 (`@nestjs/common`, `@nestjs/core`, `@nestjs/platform-express`)
- **@nestjs/cli** 10.4.9 (build)
- **node-postgres (`pg`)** 8.13.1 — connection pooling with per-request transactions
- **jose** 5.9.6 — JWT verification (HS256, pinned algorithm)
- **Zod** 3.23.8 — runtime validation on every boundary input
- **reflect-metadata** 0.2.2, **rxjs** 7.8.1
- **tsx** 4.19.2 — local dev server runner
- Monetary values stored / computed in **integer fils** (no floats)
- Pure, unit-tested compliance engines: DPMS cash-threshold + linked-transaction
  aggregation, customer risk scoring, OECD responsible-sourcing risk, goAML DPMSR XML
  draft builder

## 5. Database (`packages/db`)

- **PostgreSQL** (managed by **Supabase**)
- **Row-Level Security (RLS)** — `ENABLE` + `FORCE`, tenant-scoped, fail-closed policies
  (multi-tenant isolation enforced at the database layer, not application code)
- Composite foreign keys `(tenant_id, id)` — prevent cross-tenant child rows
- **Append-only, hash-chained audit log** — SHA-256, per-tenant sequence, advisory lock;
  tamper-evident (no UPDATE/DELETE policy + REVOKE + trigger)
- Versioned SQL migrations `0001`–`0007`
- Single-file deploy script `deploy/all.sql` (migrations + role grants + demo seed),
  generated by `scripts/build-deploy-sql.sh`
- Non-`BYPASSRLS` app role (`authenticated`) with explicit table grants

## 6. Security & Compliance

- **Supabase JWT authentication** (HS256, pinned algorithm) — `tenant_id` + `role` claims
  feed the Postgres RLS context (`request.jwt.claims`)
- **AES-256-GCM** document encryption (per-file IV + auth tag; key from `DOCUMENTS_ENC_KEY`)
- Fail-fast environment validation via a Zod env schema (fail-closed guards)
- `DEMO_MODE` — open, fixed-tenant demo auth path; deliberately separate from production
  auth and mutually exclusive with dev-header mode; blocked from combining unsafely
- No PII in logs or error messages
- **Never auto-files** a goAML report — the flow always ends at an explicit human
  "review & file" action

## 7. Integrations (`packages/integrations`)

Adapter pattern with **mock + real HTTP** implementations, selected by environment:

- **OCR / document AI** — Emirates ID extraction
- **Sanctions / PEP / adverse-media screening** — OpenSanctions-ready
- **Live gold-price feed** — item valuation
- **LLM** — Claude (goAML narrative generation)

## 8. Testing

- **Jest** 29.7.0 with **ts-jest** 29.2.5
- **@electric-sql/pglite** 0.2.17 — real PostgreSQL 16 (WASM) for live RLS + full-pipeline
  integration tests
- **supertest** 7.0.0 — HTTP e2e tests
- Coverage: `apps/api` 71 tests · `packages/db` 17 tests · `packages/integrations` 18 tests
- Golden-file tests for goAML XML; unit tests for all compliance logic; no-auto-file test

## 9. DevOps & Deployment

**Architecture:** `app.aurumtrace.com` (Vercel) → `aurumtrace-api.onrender.com` (Render) → Supabase Postgres — all on free tiers.

- **Git / GitHub** — source control (`github.com/jisanpolara/aurumtrace`, `main`)
- **Vercel** (Hobby / free) — hosts the Next.js web app
  - Root directory `apps/web`; framework auto-detected (Next.js)
  - Env-driven config (`DEMO_MODE`, `NEXT_PUBLIC_API_URL`)
- **Render** (free tier, Singapore region) — hosts the NestJS API
  - Deployed via **`render.yaml`** Infrastructure-as-Code blueprint
  - Build: `shared` → `integrations` → `api` (dependency order); pnpm installed to a
    writable prefix (Corepack unavailable on Render's read-only `/usr`); Node pinned
  - Public `/health` liveness endpoint; health-check + keep-alive target
- **Supabase** (free tier, Tokyo `ap-northeast-1`) — managed PostgreSQL
  - Connection via **Supavisor Session pooler** (IPv4, port 5432) — required because the
    direct connection is IPv6-only and Render free has no IPv6
  - Schema applied via `deploy/all.sql` in the SQL editor
- **GoDaddy DNS** — custom domain `app.aurumtrace.com`
  - CNAME → Vercel; HTTPS/TLS certificate auto-provisioned by Vercel
- **cron-job.org** — keep-alive ping to `/health` every 10 minutes (prevents free-tier
  cold starts)
- Secret management via platform environment variables (no secrets committed);
  `.env.example` templates per app
- SSL-enforced database connections for any non-local host
